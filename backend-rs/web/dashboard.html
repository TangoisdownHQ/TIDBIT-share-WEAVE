Single Agent Or Swarm?

Use one agent when the task is narrow, like review-only, summarization, or drafting under a human approval step.
Use a swarm when you want separated roles, cross-checking, policy validation, or one agent proposing while another approves.
Recommended model: one identity per agent role, never one shared token for the whole swarm.
